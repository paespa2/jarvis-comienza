#### Step 3: Configure Cloud SQL Fallback (5 minutes)
**Why**: Enables redundancy if Firebase fails (recommended for production)

**Get Connection Details**:
1. Go to: https://console.cloud.google.com/sql/instances
2. Find: `asistente-jarvis-1741893602789-instance`

**Set in Railway** (same Variables section):

### GitHub Actions (Automatic Daily Improvement)

Once database is set up, GitHub Actions will:
- **Run daily at 6 AM UTC** (configurable in `.github/workflows/daily-improve.yml`)
- **Automatically call** `/api/self-improve` endpoint
- **Analyze** last 24 hours of interactions
- **Generate improvements** (quality, relevance, coherence, completeness, emotion)
- **Auto-commit** to your branch with detailed commit message

## 📞 Troubleshooting

**If /api/health/sql-connect returns error**:
- Check FIREBASE_API_KEY is set in Railway Variables
- Verify Firebase Console shows dataconnect.yaml deployed
- Wait 2 minutes for SSL certificate to propagate

**If /api/self-improve returns 404**:
- Confirm registerSelfImproveEndpoint(app) is called in server.ts ✅ (already done)
- Check Railway has redeployed after variable changes
- Try again with fresh browser tab

**If auto-commit doesn't appear on GitHub**:
- Check GitHub Actions: https://github.com/paespa2/jarvis-comienza/actions
- Verify GITHUB_TOKEN is set in Railway (needed for auto-commits)
- Check .github/workflows/daily-improve.yml is correct
